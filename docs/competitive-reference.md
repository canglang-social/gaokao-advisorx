# 竞品与方法参考：张雪峰 & 主流志愿填报平台

> **说明（离线声明）**：本文件在离线环境下撰写，内容基于对张雪峰式建议风格与主流志愿填报产品（夸克高考、掌上高考、优志愿等）的**公开认知与合理假设**，未做实时联网核实。设计取舍据此对齐；接入真实数据/产品调研后应复核更新。

本分析在设计 UX 与推荐逻辑前完成，用于对齐三件事：**数据更新方式、信息填写方式、AI 分析方式**。

---

## 1. 张雪峰式建议的核心方法论（对产品的映射）

| 张雪峰式主张 | 在本产品中的落地 |
| --- | --- |
| **就业 > 名气**：优先看专业就业面、行业趋势、城市资源，而非只看学校光环 | 结果卡与方案均带 `employmentOutlook` 与行业趋势（`INDUSTRY_TRENDS`）；理由文案就业导向 |
| **专业、学校、城市的取舍**：不同分数段策略不同 | 档案含地域偏好（城市层级/地区）；偏好影响排序（`preferenceBonus`） |
| **冲稳保梯度 + 服从调剂**：控制滑档风险 | 核心匹配引擎输出冲/稳/保三档；理由中显式提示服从调剂、保底数量 |
| **“低分高校”/性价比**：名校处弱区位被低估 | 黑马预测（`darkHorse.ts`）专门识别此类机会，并强制风险免责声明 |
| **转专业是退路但不可依赖** | 院校带 `transfer` 难度/政策，结果卡与理由展示 |
| **理性看待预测**：分数线年年波动 | 全程新鲜度徽章 + 往年/陈旧告警；预测均附风险提示 |

## 2. 主流平台模式拆解（数据更新 / 信息填写 / AI 分析）

### 2.1 数据更新（Data updates）
- **观察/假设**：平台核心壁垒是逐年沉淀的“院校×专业×省份×年份”投档线、位次（一分一段）、招生计划，按年更新，并标注数据年份。
- **本产品对齐**：
  - 每条记录带 `year` + `lastUpdatedAt`；UI 用徽章区分“数据较新/已 N 天未更新/往年数据”。
  - 真实的**定时采集流水线**（node-cron 每日）+ 去重（自然键 upsert）+ 新鲜度统计；唯一被 mock 的是网络抓取本身。
  - 提供位次（一分一段）换算能力（`rankConversion`），对齐平台“位次优先”的匹配范式。

### 2.2 信息填写（Profile intake）
- **观察/假设**：典型入口收集 省份 / 选科(新高考) / 分数(或位次) / 兴趣 / 意向院校专业 / 地域偏好，并强调“可随时修改、即时看结果”。
- **本产品对齐**：
  - 表单覆盖以上全部字段 + 气候、自由文本想法；新高考 3+3 / 3+1+2 / 传统文理皆支持。
  - **实时联动**：任意修改 350ms 防抖后重算冲稳保；本地持久化（localStorage）。
  - 输入做归一化与边界硬化（`normalizeProfile`），缺数据省份优雅降级并提示。

### 2.3 AI 分析（AI analysis）
- **观察/假设**：新一代产品（如夸克高考的 AI 功能）以对话方式探询意图、结合趋势给“方案”，并强调结果仅供参考。
- **本产品对齐**：
  - 对话探询“就业/城市/名气”取向与风险偏好，结合 `INDUSTRY_TRENDS` 行业趋势。
  - 一键产出**稳妥型 / 均衡型 / 冲刺型**三套完整方案，每套含 items（冲稳保标注）、理由、风险。
  - 黑马横幅 + 醒目免责声明（`预测有风险,仅供参考,不可轻信。`）。
  - LLM 走可替换接口（mock / Anthropic），凭证藏在配置后。

---

## 3. 设计取舍小结

- **以“能落地的就业信息 + 风险分层”为第一价值**，而非堆砌院校数据维度。
- **新鲜度可见性**作为信任基础（对齐平台“数据年份”心智）。
- **AI 作为方案生成器与解释器**，但所有预测显著标注风险，避免“轻信”。
- **接口化一切外部依赖**（数据源/存储/LLM），保证 v0→v1 可平滑替换。

> 复核清单（接真实调研后）：核对各平台实际字段与更新频率、位次匹配算法细节、AI 方案的结构与免责合规口径，并据此微调 `matching` 阈值、`darkHorse` 启发式与 intake 字段。
