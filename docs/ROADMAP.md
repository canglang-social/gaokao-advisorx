# ROADMAP — v1 及以后

每项含：**价值 / 依赖 / 粗略工作量**（S≈数天，M≈1–2 周，L≈3–6 周，XL≈数月）。优先级大致自上而下。

## v1（让产品“可信、可用”）

### 1. 真实数据源扩省 + 爬虫合规 — 价值⭐⭐⭐⭐⭐
- **价值**：真实数据是产品的根本壁垒与可信前提。**山东 2025 综合已端到端打通**（真实，见 README 覆盖表）；待扩展到更多省份/年份。
- **内容**：把已实现的 `RealFetcher` 模式复制到其余省份——各省考试院批次线/一分一段、阳光高考院校库、投档线、学科评估、就业质量报告。
- **合规**：遵守 robots/版权与各省数据使用条款；优先官方公开数据/授权 API；限速、缓存、来源标注、错误重试与告警。
- **依赖**：数据源调研与合规评估；`DataFetcher` 接口（已就绪）。
- **工作量**：XL（合规 + 多源解析 + 数据清洗对齐）。

### 2. 真实数据库 — 价值⭐⭐⭐⭐
- **价值**：JSON 文件不适合数据量增长、并发与查询。
- **内容**：实现 `DataStore`（Postgres 首选；或 SQLite 起步），加索引（省份×年份×科类×院校）、迁移脚本；保留 `Repository` 接口不变。
- **依赖**：`DataStore` 接缝（已就绪，`sqlite` 当前为占位）。
- **工作量**：M。

### 3. 评测集扩充 + 数据质量校验 — 价值⭐⭐⭐⭐
- **价值**：真实数据上线后需回归与质量门禁。
- **内容**：评测集从 24 例扩到覆盖全部省份/批次；加入“数据完整性/异常值/年度漂移”校验作为流水线门禁。
- **依赖**：真实数据。
- **工作量**：M。

### 4. 用户账号与服务端档案 — 价值⭐⭐⭐⭐
- **价值**：跨设备同步、保存多套方案、为后续付费/追踪铺路。
- **内容**：认证（邮箱/手机/微信登录）、服务端档案存储、把 localStorage 迁移上云。
- **依赖**：真实数据库。
- **工作量**：M–L。

### 5. AI 方案质量评测对齐 — 价值⭐⭐⭐
- **价值**：真正的语言推理质量与个性化。
- **现状**：真实 AI **已可用**——`DeepSeekAiClient` / `AnthropicAiClient` 均已实现，配 key 即生效（演示默认 DeepSeek）。
- **内容**：为方案生成加结构化输出与质量评测（LLM-as-judge），把 AI 分析纳入可回归的评测。
- **依赖**：任一 LLM key；评测集。
- **工作量**：S–M。

## v1.5 / v2（增长与商业化）

### 6. 移动端 / 微信小程序 — 价值⭐⭐⭐⭐
- **价值**：高考用户主要在移动端；小程序触达强。
- **内容**：响应式优化 + 小程序壳（复用 API 层）。
- **依赖**：稳定 API。
- **工作量**：L。

### 7. 付费专家复核 — 价值⭐⭐⭐⭐（商业化核心）
- **价值**：AI 方案 + 人工专家复核是主流变现路径。
- **内容**：专家工作台、订单/支付、方案批注与回传。
- **依赖**：账号体系、支付合规。
- **工作量**：L–XL。

### 8. 录取后追踪（post-admission tracking）— 价值⭐⭐⭐
- **价值**：回收“预测 vs 实际录取”闭环，沉淀独有结果数据。
- **内容**：用户回填录取结果；构建结果数据集。
- **依赖**：账号体系。
- **工作量**：M。

### 9. 基于结果数据的模型微调 — 价值⭐⭐⭐
- **价值**：用真实录取结果提升预测线/概率/黑马的准确度。
- **内容**：以追踪数据训练/校准录取概率与位次模型；引擎阈值由数据驱动。
- **依赖**：第 8 项积累足量结果数据。
- **工作量**：L（依赖数据规模）。

## 持续技术债 / 改进

- 匹配引擎：引入分数线**趋势外推**与批次线漂移修正（当前预测线=最近一年实际线）。
- 位次匹配：从粗桶升级为逐分一分一段，提升精度。
- 可观测性：结构化日志、流水线失败告警、数据快照版本化。
- 无障碍与 i18n：当前 UI 文案为简体中文；如需多语再抽离。
