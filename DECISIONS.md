# DECISIONS — 关键技术/产品决策与权衡

记录 v0 构建过程中的主要决策。日期均为绝对日期。环境约束：离线、TypeScript、pnpm。

## 2026-06-12 · 技术栈：Next.js (App Router) + TypeScript

**决策**：用单一 Next.js 14 应用同时承载前端（React 客户端组件）与后端（API 路由 + 服务端逻辑）。
**备选**：Vite + React 前端 / 独立 Node(Express) 后端（两个进程）。
**理由**：
- 需求要求“一条命令 `pnpm install && pnpm dev` 拉起整个应用”。Next 单进程最省事，无需并行管理两个服务。
- API 路由与领域库共享 TS 类型，零序列化重复定义。
- 服务端组件/路由天然隔离 Node-only 代码（`fs`、调度器），不会泄漏进客户端 bundle。
**权衡**：Next 的约定（App Router、`'use client'`）有学习成本；定时任务需独立进程（`pnpm scheduler`）而非随 web 进程常驻——这其实更贴近生产部署形态。

## 2026-06-12 · 存储：JSON 文件 mock，藏在 `DataStore` 接口后

**决策**：默认 `JsonStore`（`data/store.json`，原子写），并定义 `DataStore` 接口 + `Repository` DAO 作为唯一存储接缝；`MemoryStore` 供测试/评测；`sqlite` 驱动留占位并显式抛“未实现”。
**备选**：直接上 SQLite（better-sqlite3）。
**理由**：题目明确“数据库连接现在是 mock”，且要求“真实 DB 可通过配置替换”。JSON 文件零原生依赖、安装最稳、便于查看；接口隔离保证未来换 Postgres/SQLite 只需实现 `DataStore` 并在 `createDataStore` 接一行。
**权衡**：JSON 全量读写不适合大数据量/并发——v0 数据量小（百级记录）完全够用，已在文档与 ROADMAP 标注为 v1 升级项。

## 2026-06-12 · 数据采集：真实流水线 + mock 抓取

**决策**：`DataFetcher` 接口抽象“网络源”，v0 只实现 `MockFetcher`（含 14 校 / 28 专业 / 多年批次线 / 一分一段表 / 投档线）。流水线（去重、按自然键 upsert、新鲜度统计）、调度器（node-cron）、新鲜度标记均为**真实**逻辑。
**理由**：题目要求“fetcher 可 mock，但调度、流水线、去重、新鲜度逻辑必须真实”。把唯一被 mock 的点收敛到 `DataFetcher` 一个接口，替换成真实爬虫/官方接口时下游零改动。
**权衡**：mock 数据为启发式构造（跨省偏移、年度趋势），并非真实分数；已在 `docs/MOCKS.md` 全量登记并标注替换来源。

## 2026-06-12 · 匹配引擎：分数差为主、位次为辅的冲/稳/保

**决策**：以 `studentScore - 预测线`（取该校在考生省份/科类的最近年份投档线）划档：`[-18,3)=冲`、`[3,20)=稳`、`≥20=保`、`<-18` 丢弃；录取概率用 logistic(差值/8)，在两端都有位次时与位次信号 60/40 融合。阈值集中为可调常量 `TIER_THRESHOLDS`。
**备选**：纯位次法（需完整一分一段对照）。
**理由**：真实平台多以位次为准，但 v0 一分一段为粗桶 mock，纯位次精度有限；分数差直观、稳健、易测，位次作为校正。引擎为**纯函数**（数据与时钟注入），便于单测与评测复用。
**权衡**：未做趋势外推（预测线=最近一年实际线），简单透明；趋势/批次线漂移修正列入 ROADMAP。

## 2026-06-12 · AI：可替换接口，默认 mock，可切 Anthropic

**决策**：`AiClient` 接口（`generatePlans` + `chat`），`MockAiClient` 确定性地由匹配结果重组三套方案；`AnthropicAiClient` 用官方 `@anthropic-ai/sdk` 调 `claude-opus-4-8` + adaptive thinking。工厂按 `AI_PROVIDER`/key 选择，缺 key 自动回退 mock。
**理由**：题目要求“用 Anthropic API，但把 key/config 藏在接口后以便替换”。mock 保证离线/无 key 也能完整演示且评测确定；真实客户端遵循官方 SDK 用法（结论先行、JSON 容错解析）。
**权衡**：mock 方案是规则重组而非真正语言推理——足以演示产品形态；真实推理质量交由切换后的 Anthropic 客户端。

## 2026-06-12 · 黑马预测：启发式 + 强制免责声明

**决策**：在 985/211/双一流中，筛选投档线下行或“名校处弱区位被低估”的院校，给出黑马指数与朴素预测线。前端横幅强制展示 `预测有风险,仅供参考,不可轻信。`
**理由**：贴合“低分高校”卖点同时控制误导风险（产品与合规要求）。
**权衡**：纯启发式，非统计模型；已在 UI/文档显著标注风险。

## 2026-06-12 · 测试/评测：vitest + tsx，相对导入

**决策**：单测用 vitest，脚本/评测用 tsx；库代码统一**相对导入**（不依赖 `@/` 别名），App 内由 Next 解析别名。
**理由**：相对导入在 tsx/vitest/Next 三处都无需额外别名配置，最稳。评测设计为独立 runner（内存库 + 真实领域逻辑 + 断言），既是 RAG 评测集也是回归网。
**权衡**：相对路径略啰嗦，换取跨工具零配置可靠性。
